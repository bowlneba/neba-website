# v0.1 Feature Plan: About & News

## Ordering Recommendation

**Do About first.** It's a handful of files with no new backend domain work. News requires a new aggregate, EF migration, two API endpoints, attachment infrastructure, two Blazor pages, and tests. About unblocks the milestone without tying up the context window.

---

## Feature 1: About Page ✅ DONE

### v0.1 Scope

A static Blazor page. Content (mission, history, board members, etc.) is hardcoded in the component. Updating it requires a PR — acceptable given how rarely it changes. Sponsors section is **live data** via the existing `ListActiveSponsors` endpoint, filtered to Title Sponsor + Premier tiers.

### Future option (deferred)

An admin user with the right role clicks a pencil icon, gets an inline rich-text editor (e.g., Trix or Quill), and saves. Backend would store the HTML blob in Azure Blob Storage under a well-known key and serve it from `GetDocument`. This follows the existing Documents pattern but adds an admin-write path. **Out of scope for v0.1.**

### Page structure — **Option A selected**

Gradient hero banner → three pillars → mission/history two-column → board of directors grid → sponsors strip with "View all sponsors" link.

See `docs/plans/about-layout-options.html` for the rendered mockup.

### API changes

None. Reuse existing `GET /sponsors` (ListActiveSponsors). Filter in the Blazor component: keep sponsors where `Tier == "Title Sponsor" || Tier == "Premier"`.

## Feature 2: News

### v0.1 Scope

- Public read-only views: article list (`/news`) and article detail (`/news/{slug}`)
- Domain model + EF migration (no data yet — admin write UI is a future milestone)
- Article Attachments and Header Images stored in Azure Blob Storage
- Home page: **remove** the news section for v0.1

### Domain model

#### `Article` aggregate (`Neba.Api.Features.News.Domain`)

| Property | Type | Notes |
|---|---|---|
| `Id` | `ArticleId` (ULID) | Domain identity |
| `Title` | `string` | Required, max 300 |
| `Slug` | `string` | URL-safe, unique across articles |
| `Content` | `string` | HTML body (stored as-is) |
| `PublicationStatus` | `PublicationStatus` | SmartEnum: `Draft` or `Published`. Defaults to `Draft` |
| `PublishDateUtc` | `DateTimeOffset` | When the article becomes publicly visible. If not provided at publish time, defaults to `DateTime.UtcNow` (published immediately) |
| `HeaderImage` | `StoredFile?` | Nullable — not every article has an image |
| `TournamentId` | `TournamentId?` | Optional reference to a related Tournament |
| `_attachments` | `List<ArticleAttachment>` | Child collection (backing field) |

An Article is publicly visible only when `PublicationStatus == Published && PublishDate <= UtcNow`.

#### `ArticleAttachment` child entity (`internal static ErrorOr<ArticleAttachment> Create(...)`)

An article can have zero or more attached files (e.g., a lane pattern PDF, a results sheet).

| Property | Type | Notes |
|---|---|---|
| `Id` | `ArticleAttachmentId` (ULID) | |
| `DisplayName` | `string` | Staff-provided label shown in the attachment list (e.g., "Finals Bracket") |
| `File` | `StoredFile` | Container, Path, ContentType, SizeInBytes |
| `IsInline` | `bool` | Whether this attachment is also embedded in the article Body. Defaults to `false` |

Blob path conventions: Header images at `news/{articleId}/header/{filename}`; attachments at `news/{articleId}/attachments/{filename}`.

`StoredFile` mapping details (nullable header image, owned entity vs. complex property) will be resolved at implementation time.

### Database schema

**Table: `articles`**
- `id` int identity — shadow `db_id`
- `domain_id` char(26) — AK1, maps to `ArticleId`
- `title` varchar(256)
- `slug` varchar(256) — unique index
- `content` text
- `publication_status` integer — SmartEnum int value (0=Draft, 1=Published)
- `publish_date_utc` datetimeoffset not null
- `header_image_*` — `StoredFile` complex property (nullable; EF 10 nullable complex property)
- `tournament_id` char(26) nullable — FK → `tournaments.domain_id`
- Index: `(publication_status, publish_date_utc)` (list query filter + sort)

**Table: `article_attachments`**
- `id` int identity — shadow `db_id`
- `domain_id` char(26) — AK
- `article_id` int — FK → `articles.id`
- `display_name` varchar(128)
- `is_inline` bit not null
- `file_*` — `StoredFile` complex property (required)

### API endpoints

#### `GET /news` — List published articles

**Query**: `ListArticlesQuery`  
**Response**: `IReadOnlyCollection<ArticleSummaryDto>`  
**DTO**: `{ Slug, Title, Excerpt, HeaderImageUrl, PublishDate }` — `Excerpt` is a truncated `Body` produced by the query handler; not a stored domain property  
**Filter**: `PublicationStatus == "Published" && PublishDate <= now`  
**Order**: `PublishDate DESC`  
**Pagination**: `?page=1&pageSize=10` — 10 articles per page, required day one

#### `GET /news/{slug}` — Get single article

**Query**: `GetArticleQuery(Slug)`  
**Response**: `ArticleDetailDto`  
**DTO**: `{ Slug, Title, Body, HeaderImageUrl, PublishDate, Attachments: [{ DisplayName, FileUrl }] }`  
**404** if not found or not published

Both endpoints are `AllowAnonymous()`.

- Two-column layout: main content area (flexible width) + 280px sticky sidebar.
- Sidebar sticks at `top: 5rem` (below navbar) as the user scrolls the article body.
- Header image is a contained rounded card (aspect-ratio 16/7) — absent entirely if `HeaderImage` is null.
- Sidebar "Article Info" panel: just published date for v0.1 (room to add author/tags later).
- Sidebar "Attached Files" panel: hidden if `Attachments` is empty; each row has icon, display name, download link.
- On mobile: sidebar moves below the article body; image is full-width at top.
- `← Back to News` appears both above the image and below the article body.
